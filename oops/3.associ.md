
# 1. Inheritance → IS-A

## Meaning

One class becomes a specialized version of another class.

```text
Dog IS-A Animal
```

---

# Simplest Syntax

```java
class Parent {
}

class Child extends Parent {
}
```

`extends` = inheritance.

---

# Real Practical Example

## Parent Class

```java
class Animal {

    void sound() {
        System.out.println("Animal makes sound");
    }
}
```

---

## Child Class

```java
class Dog extends Animal {

    void bark() {
        System.out.println("Dog barks");
    }
}
```

---

## Main Method

```java
public class Main {

    public static void main(String[] args) {

        Dog d = new Dog();

        d.sound(); // inherited method
        d.bark();  // own method
    }
}
```

---

# Output

```text
Animal makes sound
Dog barks
```

---

# What Happened?

Dog automatically got:

```java
sound()
```

from Animal.

Because:

```text
Dog IS-A Animal
```

# 2. Association → USES-A

## Meaning

One class simply uses another class.

---

# Real Example

```text
Student USES Pen
```

Student is not a Pen.

Student just uses it.

---

# Simple Code

## Pen Class

```java
class Pen {

    void write() {
        System.out.println("Writing...");
    }
}
```

---

## Student Class

```java
class Student {

    void study() {

        Pen p = new Pen();

        p.write();
    }
}
```

---

## Main

```java
public class Main {

    public static void main(String[] args) {

        Student s = new Student();

        s.study();
    }
}
```

---

# Output

```text
Writing...
```

---

# Why Association?

Student only needs Pen functionality.

No inheritance needed.

---

# 3. Aggregation → HAS-A


Aggregation means:

```text id="1u3m4o"
One class HAS-A another class object
```

---

# Basic Syntax

```java id="c4gx7j"
class A {

    B b;
}
```

Here:

* class `A` contains object of class `B`

So:

```text id="6lyu1f"
A HAS-A B
```

---

# Real Example

## Teacher Class

```java id="3efjiv"
class Teacher {

    String name;

    Teacher(String name) {
        this.name = name;
    }
}
```

---

## School Class

```java id="uzfxq7"
class School {

    Teacher teacher;

    School(Teacher teacher) {

        this.teacher = teacher;
    }
}
```

---

# Main Method

```java id="m0wrgu"
public class Main {

    public static void main(String[] args) {

        Teacher t = new Teacher("Ramesh");

        School s = new School(t);
    }
}
```

---

# Understand Important Line

```java id="zvzk1v"
Teacher teacher;
```

This line means:

```text id="x7g1jt"
School HAS-A Teacher
```

That is aggregation.

---

# Real-Life Examples

```text id="j5v5se"
Car HAS-A Engine
Library HAS-A Books
Department HAS-A Employees
School HAS-A Teachers
```

---

# Key Point

Object is stored as class variable:

```java id="th7snq"
Engine engine;
```

That is the main aggregation syntax.


# Most Important Syntax Difference

---

# Inheritance

Uses:

```java
extends
```

Example:

```java
class Dog extends Animal
```

---

# Association

Usually:

```java
ClassA uses ClassB
```

inside methods or variables.

Example:

```java
Pen p = new Pen();
```

---

# Aggregation

One class stores another object.

Example:

```java
Teacher teacher;
```

inside another class.

---

# Very Easy Memory Trick

---

# Inheritance

```text
IS-A
```

Example:

```text
Dog IS-A Animal
```

Uses:

```java
extends
```

---

# Association

```text
USES-A
```

Example:

```text
Student USES Pen
```

Uses:

* method calls
* object creation

---

# Aggregation

```text
HAS-A
```

Example:

```text
Department HAS-A Teacher
```

Uses:

* object as field/instance variable
